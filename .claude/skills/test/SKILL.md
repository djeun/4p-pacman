4p-pacman 로컬 테스트 절차를 수행하라.

## 1. 서버 실행 전 확인

- `package.json` 에 `dev` 스크립트 존재 여부 확인
- `node_modules/` 존재 여부 확인 → 없으면 `npm install` 먼저 실행
- 포트 충돌 확인: `netstat -ano | findstr :3000` (Windows)

## 2. 서버 실행

```bash
npm run dev
```

서버가 정상 시작되면 출력 로그에서 포트 번호 확인 후 사용자에게 알린다.

## 3. 멀티플레이어 테스트 절차 안내

사용자에게 아래 절차를 안내한다:

**브라우저 탭 다중 접속 테스트**
1. `http://localhost:<PORT>` 를 탭 4개에서 열기
2. 탭 1: 방 생성 → 룸 코드 확인
3. 탭 2~4: 룸 코드 입력 → 참가
4. 모든 플레이어 준비 → 게임 시작

**테스트 시나리오**
- [ ] 방 생성 및 참가 정상 동작
- [ ] 4명 모두 각 코너에서 시작
- [ ] 공격 동전 생성 확인 (랜덤 위치)
- [ ] 동전 먹기 → 빨간색 전환 + 타이머 게이지 확인
- [ ] 빨간 팩맨이 흰색 팩맨 접촉 → 사망 처리
- [ ] 바운스 백 동작 확인
- [ ] 라운드 종료 및 점수 계산
- [ ] 관전 모드 전환

## 4. 오류 발생 시

서버 로그에서 오류 메시지를 확인하고 원인을 진단한다.
Socket.io 연결 오류 시 CORS 설정을 먼저 확인한다.
